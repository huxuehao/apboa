# 技能包管理

技能包是工具的高阶组合形态，将一组相关工具、资源引用、示例和脚本打包为可复用的技能单元，赋予智能体完成特定领域复杂任务的能力。


## 功能概述

- 支持按分类管理技能包
- 技能包可关联多个工具，运行时自动注册到工具集
- 支持资源引用、示例和脚本三种扩展内容
- 支持三种导入模式：本地装载、ZIP 压缩包、Git 仓库
- 导入时支持覆盖/跳过策略，同名技能包可灵活处理
- 修改技能包时自动装载脚本到本地


## 核心概念

### 技能包组成

一个技能包由以下部分组成：

| 组成 | 说明 | 格式 |
|------|------|------|
| **基本信息** | 名称、描述、分类 | 文本 |
| **技能内容** | 技能概述，描述该技能的能力和使用方式 | Markdown |
| **资源引用** | 参考文档、知识片段等 | JSON 数组 `[{name, content}]` |
| **示例** | 技能使用的示例说明 | JSON 数组 `[{name, content}]` |
| **脚本** | 可执行的脚本文件 | JSON 数组 `[{name, content}]` |
| **关联工具** | 技能包依赖的工具列表 | 工具 ID 列表 |

### 技能包与工具的关系

- 技能包可以关联 0~N 个工具
- 运行时，技能包关联的工具会自动注册到智能体的工具集中
- 一个工具可以被多个技能包关联

### 导入方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **本地装载** | 指定本地目录路径，直接装载目录中的技能包 | 已有技能包目录、开发调试 |
| **ZIP 导入** | 上传 ZIP 压缩包，自动解压并导入 | 离线分享、批量导入 |
| **Git 导入** | 从 Git 仓库拉取技能定义 | 版本管理、团队协作 |
| **手动创建** | 在线填写技能包信息 | 简单技能、快速创建 |


## 操作指南

### 手动创建技能包

1. 在技能包管理页面点击 **添加新技能包** 卡片
2. 填写表单：
   - **分类**：选择或新建分类
   - **名称**：技能包名称
   - **描述**：技能功能说明
   - **技能内容**：使用 Markdown 编辑器编写技能概述
   - **资源引用**：添加参考文档（名称+内容键值对）
   - **示例**：添加使用示例
   - **脚本**：添加可执行脚本
   - **关联工具**：选择技能包依赖的工具
3. 点击确认提交

### 导入技能包

1. 点击导入按钮
2. 选择导入方式：
   - **本地装载**：填写本地技能包目录路径
   - **ZIP 导入**：上传 ZIP 文件
   - **Git 导入**：填写仓库地址
3. 选择覆盖策略：
   - **跳过**：遇到同名技能包时保留已有内容，跳过导入
   - **覆盖**：遇到同名技能包时删除旧内容，导入新内容
4. 系统自动解析并安装技能包

### 编辑技能包

点击卡片右上角菜单 → **编辑**，修改技能包内容和关联工具。

:::warning
修改技能包后，关联智能体将自动重新注册。脚本变更会自动重新装载到本地。
:::

### 搜索与筛选

- **分类筛选**：页面顶部分段控制器切换分类
- **名称搜索**：输入关键词后回车搜索


## 在智能体中使用

在智能体配置的「工具与能力」步骤中：

1. 在 **技能包** 区域，按分类折叠面板浏览
2. 勾选需要的技能包（Checkbox 多选）
3. 技能包关联的工具会自动注册到智能体的工具集中

## 最佳实践

### 技能内容编写

技能内容是模型理解技能能力的关键，建议包含：

- **技能目标**：该技能解决什么问题
- **使用场景**：什么情况下应该使用该技能
- **操作步骤**：使用该技能的典型步骤
- **注意事项**：使用限制和注意点

### 工具关联建议

- 将逻辑上相关的工具组合到同一技能包
- 避免将不相关工具放入同一技能包
- 确保技能包关联的工具都已启用

### 脚本管理

- 导入技能包时，整个技能包目录（含脚本、示例、引用）会安装到平台工作空间
- 手动编辑技能包时，脚本会自动装载到本地文件系统
- 清空脚本内容时，本地脚本文件会自动移除
- 脚本文件路径遵循平台的命名规范


## 导入模式详解

### 本地装载

本地装载模式用于从服务器本地目录直接导入技能包。适用于已有技能包目录或开发调试场景。

**工作原理：**

1. 读取指定目录下的技能包定义文件（`skill.md`）
2. 解析技能包的名称、描述、内容以及资源（脚本、示例、引用）
3. 将技能包目录安装到平台工作空间的 `skills/` 目录下
4. 将技能包元数据持久化到数据库

**目录结构要求：**

```
<指定路径>/
  skill-a/
    skill.md          # 技能定义文件
    scripts/           # 脚本文件
    examples/          # 示例文件
    references/        # 引用文件
  skill-b/
    skill.md
    ...
```

如果指定路径下存在 `skills/` 子目录，系统会优先从该子目录加载技能包。

:::info
当本地路径与平台工作空间路径相同时，系统会跳过文件复制，仅更新数据库记录，避免不必要的文件操作。
:::

### ZIP 压缩包导入

ZIP 导入模式用于通过上传压缩包批量导入技能包。适用于离线分享和批量部署场景。

**工作原理：**

1. 将上传的 ZIP 文件解压到平台临时目录（`.apboa/temp/`）
2. 读取解压后目录中的技能包定义
3. 根据覆盖策略将技能包目录安装到工作空间
4. 安装完成后自动删除临时目录

**ZIP 包目录结构要求：**

```
skills/
  skill-a/
    skill.md
    scripts/
    examples/
    references/
  skill-b/
    skill.md
    ...
```

ZIP 压缩包的根目录下必须包含 `skills/` 子目录，各技能包以子目录形式组织在其中。

:::warning
ZIP 导入过程中会创建临时目录用于解压，导入完成后（无论成功或失败）临时目录会被自动清理，不会占用额外磁盘空间。
:::

### Git 仓库导入

Git 导入模式用于从远程 Git 仓库拉取技能包。适用于版本管理和团队协作场景。

**工作原理：**

1. 将远程仓库克隆到平台临时目录（`.apboa/temp/`）
2. 如果仓库根目录下存在 `skills/` 子目录，优先从该子目录加载；否则从仓库根目录加载
3. 根据覆盖策略将技能包目录安装到工作空间
4. 安装完成后自动删除临时目录

**仓库目录结构要求：**

```
<仓库根目录>/
  skills/              # 推荐：技能包目录
    skill-a/
      skill.md
      scripts/
      examples/
      references/
    skill-b/
      ...
  README.md
  ...
```

:::info
如果仓库中不存在 `skills/` 子目录，系统会将仓库根目录视为技能包目录，直接从根目录加载技能包。
:::


## 覆盖策略说明

导入技能包时，需要选择覆盖策略来处理同名技能包冲突：

| 策略 | 行为 | 适用场景 |
|------|------|----------|
| **跳过** | 同名技能包保留已有内容，跳过导入 | 防止误覆盖、保留本地修改 |
| **覆盖** | 删除同名技能包的旧目录和数据库记录，导入新内容 | 版本更新、全量同步 |

**覆盖策略的影响范围：**

- **文件层面**：跳过时保留已有目录，覆盖时先删除旧目录再安装新目录
- **数据库层面**：跳过时不更新数据库记录，覆盖时更新技能包元数据

:::warning
选择覆盖策略时需谨慎。覆盖操作会删除旧技能包的整个目录（包括手动添加的文件），再安装新技能包。如果旧技能包中有手动修改的内容，请先备份。
:::


## 导入建议

### 根据场景选择导入方式

- **开发调试**：使用本地装载模式，直接指向开发目录，修改后可快速重新导入验证
- **离线分享**：将技能包目录打包为 ZIP 文件，接收方通过 ZIP 导入安装
- **团队协作**：将技能包纳入 Git 仓库管理，团队成员通过 Git 导入同步最新版本
- **快速创建**：对于简单的技能包，可直接手动创建

### 覆盖策略选择建议

- **首次导入**：无需关注覆盖策略，两种均可
- **增量导入**：选择跳过策略，避免覆盖已有技能包的本地修改
- **版本更新**：选择覆盖策略，确保技能包内容与源同步
- **不确定时**：选择跳过策略更安全，可逐个确认后再覆盖导入

### 目录结构规范

为了确保导入顺利，建议遵循以下目录结构规范：

1. 每个技能包使用独立目录，目录名即为技能包名称
2. 技能定义文件统一命名为 `skill.md`
3. 脚本文件放在 `scripts/` 子目录
4. 示例文件放在 `examples/` 子目录
5. 引用文件放在 `references/` 子目录
6. 多个技能包统一放在 `skills/` 父目录下

### 导入后验证

导入完成后，建议进行以下检查：

1. 在技能包管理页面确认技能包列表已更新
2. 点击技能包查看详情，确认内容完整
3. 在智能体中关联该技能包，验证运行时工具注册是否正常
