# 模型供应商管理

模型供应商是智能体的"推理引擎"，负责管理不同 AI 服务提供商的连接信息和认证配置。通过统一的供应商抽象层，平台支持接入多种主流大模型服务。


## 功能概述

- 支持 4 种供应商类型：DashScope、OpenAI、Anthropic、Ollama
- 每个供应商下可配置多个模型
- 支持两种认证方式：直接配置 `API Key`、`环境变量`引用
- 供应商下的模型可独立管理，支持启用/禁用


## 核心概念

### 供应商类型

| 类型 | 说明 | 默认 Base URL |
|------|------|---------------|
| **DashScope** | 阿里云百炼平台 | `https://dashscope.aliyuncs.com` |
| **OpenAI** | OpenAI 官方 API | `https://api.openai.com` |
| **Anthropic** | Anthropic Claude API | `https://api.anthropic.com` |
| **Ollama** | 本地 Ollama 服务 | `http://localhost:11434` |

### 认证方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **直接配置（CONFIG）** | 在平台上直接填写 API Key | 快速配置，Key 加密存储 |
| **环境变量（ENV）** | 填写环境变量名，运行时从环境变量读取 | 更安全，适合生产环境 |

### 模型配置

每个供应商下可配置多个模型，每个模型包含：

| 属性 | 说明 |
|------|------|
| 模型名称 | 显示名称 |
| 模型编号 | 模型标识符（如 `gpt-4o`、`qwen-max`） |
| 模型类型 | 文本 / 图像 / 音频 / 视频 |
| 是否支持流式 | 是否支持流式输出 |
| 是否支持思考 | 是否支持思考模式（深度推理） |
| 上下文窗口 | 最大上下文 Token 数 |
| 最大输出 | 最大输出 Token 数 |
| 温度 | 生成随机性参数（0~2） |
| Top-P | 核采样参数 |
| Top-K | Top-K 采样参数 |
| 重复惩罚 | 重复内容惩罚系数 |
| 随机种子 | 可复现生成的种子值 |


## 操作指南

### 创建供应商

1. 在模型供应商管理页面点击 **添加模型供应商** 卡片
2. 填写表单：
   - **类型**：选择供应商类型（选择后自动填充默认 Base URL）
   - **名称**：供应商名称
   - **描述**：供应商说明
   - **Base URL**：API 地址
   - **认证方式**：直接配置或环境变量
   - **API Key / 环境变量名**：根据认证方式填写
3. 点击确认提交

### 管理模型

1. 在供应商卡片右上角菜单 → **配置模型**，打开模型管理弹窗
2. 可执行以下操作：
   - **新增模型**：填写模型名称、编号、类型和参数
   - **编辑模型**：修改模型配置
   - **启用/禁用模型**：控制模型可用性
   - **删除模型**：移除模型配置

:::warning
删除或修改被智能体使用的模型时，关联智能体将自动重新注册。
:::

### 查看供应商详情

点击卡片右上角菜单 → **查看**，可查看供应商完整信息。

### 删除供应商

点击卡片右上角菜单 → **删除**。

:::warning
删除供应商前需确认其下没有被智能体使用的模型。系统会进行二次确认。
:::

### 搜索与筛选

- **类型筛选**：页面顶部分段控制器切换供应商类型
- **名称搜索**：输入关键词后回车搜索


## 在智能体中使用

在智能体配置的「模型与提示词」步骤中：

1. 选择供应商（按类型分类展示）
2. 选择该供应商下的模型
3. 可选配置模型参数覆盖（温度、Top-P 等）

## 常见问题

### 如何接入 Ollama 本地模型？

1. 创建类型为 `Ollama` 的供应商
2. Base URL 填写 Ollama 服务地址（默认 `http://localhost:11434`）
3. 认证方式选择"直接配置"，API Key 可留空或填写任意值
4. 在模型管理中添加本地已下载的模型编号

### 如何接入兼容 OpenAI 协议的第三方服务？

1. 创建类型为 `OpenAI` 的供应商
2. 修改 Base URL 为第三方服务地址
3. 填写对应的 API Key

### 环境变量认证如何使用？

1. 认证方式选择"环境变量"
2. 填写环境变量名（如 `OPENAI_API_KEY`）
3. 在部署环境中设置对应的环境变量值
4. 系统运行时自动从环境变量读取 API Key
